## 🧠 Concept Quest — Code Review
Real programmers read each other's code and explain it. Today, **you** review your own code:

- Read your program **block by block**, out loud. Does each part make sense?
- Add **`#` comments** above the tricky parts explaining *what* they do and *why*. Future readers (including future you!) will thank you.

```python
# roll two dice and add them up for the player's starting hand
player_total = random.randint(1, 6) + random.randint(1, 6)
```

Good comments explain the *why*, not the obvious *what*. `x = x + 1  # add one` isn't helpful; `score = score + 1  # reward a correct answer` is.
