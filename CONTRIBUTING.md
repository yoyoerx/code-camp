# Contributing

Thanks for your interest! **Code Catalyst** is a personal, self-guided Python curriculum built for one rising 6th grader — but you're very welcome to use, adapt, and improve it.

## Spotted a typo, bug, or have an idea?
- **Open an [issue](https://github.com/yoyoerx/code-camp/issues)** describing the problem or suggestion. Small notes are fine — "the Day 9 solution has a typo" is plenty.
- For a fix you'd like to contribute directly: fork the repo, make your change on a branch, and open a **pull request**.

## A few things to keep in mind
- **Audience first.** This is written *for an 11–12-year-old working mostly independently.* Keep explanations friendly, concrete, and jargon-light.
- **Keep the structure.** Each day is one folder with a `README.md` (the lesson), a `starter.py` (stubs + `# TODO`s), and a `solution/` answer key. New material should follow that pattern.
- **Test starter and solution files** by running them (`python starter.py`) before submitting. The one intentional exception is `day-08-debugging/starter.py`, which contains deliberate bugs for the student to fix.
