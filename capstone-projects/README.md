# 🏗️ Capstone Projects (Week 6)

This is the big one. Over the last 4 days of camp, you'll build **one complete program from scratch** — no `solution/` folder this time, because *you* are the author now. You've got every skill you need: variables, input, decisions, loops, lists, dictionaries, and functions.

## Pick ONE path

Read all three blueprints, then choose the one that excites you most. Each blueprint gives you the **idea, a feature checklist, a suggested build order, and stretch goals** — but *how* you build it is up to you.

| Path | Theme | Best if you like… | Difficulty |
|---|---|---|---|
| **[A — Virtual Pet](path-a-virtual-pet.md)** 🐾 | Keep a digital pet happy & alive | Cute stuff, caretaking, dictionaries | ⭐ Friendliest start |
| **[B — Space Cargo Trader](path-b-space-trader.md)** 🚀 | Buy low, sell high across planets | Strategy, money, math | ⭐⭐ Medium |
| **[C — Blackjack Casino](path-c-blackjack.md)** 🃏 | Card game vs. the computer | Cards, rules, logic puzzles | ⭐⭐⭐ Spiciest |

## How the 4 days are spent

- **Day 21 — Plan it.** Choose your path. **Draw a flowchart on paper** of how the game flows. Set up the main menu loop and your starting variables.
- **Day 22 — Build the engine.** Write *pseudocode comments first*, then the real code. Get a rough, ugly, *working* version running.
- **Day 23 — Polish it.** Title screen, ASCII art, `time.sleep()` for drama, and make sure weird inputs don't crash it.
- **Day 24 — Showcase!** Add `#` comments, do a code review, present it, and let family play it.

## 🧑‍🏫 Mentor Checkpoint — before you write any code (Day 21)

The #1 mistake is picking something **too big to finish**. Talk it through with your mentor:
> "What is the *simplest version* of this that still works and is fun?" Build **that** first. You can always add more.

## The "definitely include these" checklist (for any path)

Your finished capstone should use at least:
- [ ] **Variables** to track state (score, health, money…)
- [ ] **A main loop** (`while`) that keeps the game going
- [ ] **`if`/`elif`/`else`** decisions
- [ ] **A list or dictionary** to store your data
- [ ] **At least 2 functions** you wrote yourself
- [ ] **`input()`** that doesn't crash on weird answers
- [ ] **A way to win or lose** (an ending!)

Good luck, author. Make something *you* would want to play. 🌟
