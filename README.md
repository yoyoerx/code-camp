# 🚀 Welcome to Code Catalyst!

Hi future programmer! This is your **6-week Python summer camp**. By the end you will have built games, tools, and one big final project that you designed yourself. 😎

You can do almost all of this **on your own** — that's the fun part. But you also have a grown-up "mentor" who checks in with you twice a day. Asking them for help is part of the plan, not a sign you did anything wrong.

---

## 🧭 How each day works

Every day is in its own folder, like `week-1-variables-and-io/day-01-print/`. Inside each day folder you'll find:

| File / folder | What it is |
|---|---|
| `README.md` | **Read this first.** It's the whole lesson for the day. |
| `starter.py` | The code file. It has examples to type along with, then a challenge with `# TODO` notes telling you what to add. |
| `solution/` | The finished answer. Only open it **after** you've really tried (see the rule below). |

> 📖 **Tip:** Read each `README.md` using VS Code's **Markdown Preview** — open the file and press **`Ctrl + Shift + V`** (or `Ctrl + K` then `V` for side-by-side). That makes it pretty *and* makes the hint drop-downs clickable. See `SETUP.md` Step 6.

**Your daily routine (about 2 hours):**

1. ⭐ **Kickoff (with your mentor)** — Read today's *Goal* and do the Warm-Up puzzle together.
2. 🧠 **Concept Quest** — Read the lesson and watch the short video link. Say the idea out loud in your own words.
3. ⌨️ **Guided Practice** — Open `starter.py` and type the example code yourself. Run it. Fix any errors.
4. 🛠️ **Build Block** — Finish the `# TODO`s to build the day's project. Stuck? Use the **Hints** at the bottom of the README.
5. ⭐ **Show & Tell (with your mentor)** — Run your project for your mentor and explain what you made.

---

## ▶️ How to run your code

1. Open the day's `starter.py` in VS Code.
2. Click the **▶ Run** button in the top-right corner (or press **F5** and choose "Run Without Debugging").
3. Look at the **Terminal** panel at the bottom — that's where your program talks to you and where you type answers.

If the Run button isn't there, an adult should check **[SETUP.md](SETUP.md)** with you.

---

## 🆘 The "Stuck?" Rule

Getting stuck is **normal** — it happens to every programmer, every day. Here's the plan when it happens:

1. **Read the error.** Python prints red text with a **line number**. Go look at that line.
2. **Try for ~15 minutes.** Reread the lesson. Check the **Hints** at the bottom of the README (they reveal a little at a time).
3. **Peek at the `solution/`.** This is *allowed*. Read it, understand *why* it works, then close it and retype it yourself.
4. **Ask your mentor.** Saying "here's what I tried and here's the error" is exactly what real engineers do.

> 🔑 Looking at the answer to *understand* it is **learning**. Copying it without understanding is the only way to "cheat" — and it only cheats yourself out of the fun.
